Static Gen

The simple, generic, JVM templating system
Usage Maven Documentation Github 
$ cat files.yaml
copy:                               # <-- Copy these
    - css/style.css                 #     files
    - js                            # <-- or directories
    - vendor
model:
    license: MIT
files:                              # <-- Generate these files
    /:                              # <-- Files that end in / are mapped to index.html
        view: home.html             # <-- The pebble template file to render
        model:                      # <-- Add variables to the rendering
            title: Static Gen       # <-- This will be available as {{'{{ title }}'}}

    /articles/{{'{{file}}'}}.html:  # <-- File names can be templatized
        view: article.html
        fileModels: "data/articles" # <-- The file will be generated for all items within this file

Features

  • CLI application
  • Maven plugin
  • Reading models from markdown files
  • Extracting YAML model data within markdown file headers
  • Reading models from JSON files
  • Reading models from YAML files

Command Line

Static Gen can be used from the command line after installing it.

$ ./sg --help
Static Gen.

Usage:
  sg [--manifest=FILE] [--source=DIR] [--out=DIR] [--watch] [--serve [--port=PORT]]
  sg (-h | --help)
  sg --version

Options:
  -h --help                  Show this screen.
  --version                  Show version.
  -m FILE --manifest=FILE    Manifest file [default: files.yml].
  -o DIR --out=DIR           Output directory [default: dist].
  -i DIR --source=DIR        Input directory [default: src].
  -w --watch                 Watch for input changes [default: false].
  -s --serve                 Serve the output directory [default: false].
  -p PORT --port=PORT        Port to serve on [default: 3000].
